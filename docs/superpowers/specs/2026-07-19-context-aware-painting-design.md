# 배치 우선 + 배경 보며 칠하기 + 스포이드 페널티 — 설계

## 배경 및 목표

현재 제작 흐름은 업로드 → **색칠**(인형만 단독 256×256 캔버스) → **배치**(위치 결정) → 테스트 순서다. 색을 고를 유일한 방법은 스포이드(사진을 탭하면 즉시 정확한 색을 추출)뿐이라, 제작자가 실제 배치될 자리의 배경을 보지 못한 채 칠하거나 스포이드로 완벽한 색만 뽑아 쓰게 된다. 실제로 테스트 플레이해본 결과 스포이드로 정확히 추출한 색은 위장이 지나치게 완벽해져 찾는 사람의 난이도가 과도하게 올라가는 문제가 확인됐다.

목표는 두 가지다.
1. 메챠카멜레온처럼 **인형이 실제로 놓일 자리의 배경을 보면서** 색을 칠할 수 있게 한다.
2. 스포이드(완벽한 색 추출)는 계속 허용하되, 기본값은 "제작자가 감으로 색을 고르는 것"이 되도록 유도하고, 스포이드를 쓰면 페널티가 따르게 한다.

## 범위 밖 (별도 스펙으로 분리)

- **다중 인형(1~3개) 지원**: 배치·판정·힌트·등급·공유카드·데이터 모델 전반에 영향을 주는 별개 기능이라 이 스펙에는 포함하지 않는다. 논의 중 합의된 방향성만 기록해둔다 — 인형이 여러 개면 전부 찾아야 클리어, 인형마다 위치·색을 독립적으로 정한다(따로 배치 → 따로 칠하기를 인형 수만큼 반복). 후속 스펙에서 다룬다.
- **투명 인형 버그 수정**: `renderStage()`의 표시 조건에 `'play'` 단계가 빠져 있어 실제 플레이 중 인형이 전혀 렌더링되지 않던 문제. 이 스펙과 무관한 회귀 버그라 이미 별도로 수정·배포 완료했다(commit `dcdb761`).

## 1. 제작 흐름 순서 변경

**변경 전**: 업로드 → 색칠 → 배치 → 테스트
**변경 후**: 업로드 → **배치** → **칠하기** → 테스트

배치를 먼저 해야 칠하기 단계에서 "그 자리의 배경"을 알 수 있다. 배치 화면의 "다시 색칠" 버튼은 그대로 유지하되 이제는 칠하기 화면 → 배치 화면으로 돌아가는 버튼이 되고, 칠하기 화면에는 "다시 배치"가 추가되어 배치 화면으로 돌아갈 수 있다. 배치를 바꾸고 칠하기로 다시 오면 배경 미리보기(아래 2번)만 새로 캡처되고, 이미 칠한 인형의 색 데이터(`paintCanvas`)는 별도 레이어라 그대로 유지된다.

기존 "테스트 플레이" 버튼(로컬 test 페이즈)은 배치 정보가 이미 확정된 이후 단계이므로 위치상 변화 없음.

## 2. 칠하기 화면 — 배경을 보며 칠하기

- 칠하기 화면 진입 시, 확정된 배치(`state.doll`: x, y, scale, rotation, flip)를 기준으로 사진에서 인형 주변 영역을 크롭해 정적 배경 이미지로 캡처한다. 크롭 범위는 인형 바운딩 크기의 약 1.6배로 잡아, 인형뿐 아니라 주변 배경이 여유 있게 보이도록 한다.
- **배경은 고정, 인형은 회전**: 배경 크롭은 원본 사진 방향 그대로(회전 없이) 표시하고, 그 위에 인형을 실제 배치 각도(`rotation`)·좌우반전(`flip`)만큼 회전시켜 그린다. 이는 `renderStage()`가 이미 배치/플레이 화면에서 쓰는 `translate → rotate → (flip) → drawImage` 변환과 동일한 방식이라 기존 로직을 재사용할 수 있다.
- 배경 캡처는 **정적 스냅샷**이다 — 칠하기 화면 진입 시 1회만 캡처하고, 칠하는 동안 다시 계산하지 않는다. 저장되는 데이터는 지금과 동일하게 인형 PNG(`dollComposite`/`encodeDollB64`)뿐이며 배경 크롭 자체는 저장하지 않는다.
- 브러시/지우개/실행취소/전체지우기와 인형 실루엣 클리핑 로직은 변경 없음. 화면에 배경이 추가로 깔리는 것과, 붓 입력 좌표를 인형의 로컬(비회전) 256×256 좌표계로 변환하는 부분만 새로 필요하다 — 화면에 표시된 회전된 인형 위의 터치 좌표를 `-rotation`만큼 역회전시켜 기존 `paintCtx` 좌표계로 매핑한다.

## 3. 색상 선택 — 브라우저 네이티브 피커가 기본값

- 스와치 옆에 `<input type="color">`를 추가한다. 제작자가 색상환에서 직접 고르면 그 색이 `state.color`가 된다. 외부 라이브러리 없이 구현 가능하고, 전체 RGB 공간을 자유롭게 탐색할 수 있어 "감으로 고르기"에 적합하다.
- 브러시로 칠하는 동작 자체는 지금처럼 캔버스를 탭/드래그하면 바로 적용된다. 기존의 "사진을 탭하면 자동으로 색 추출" 동작은 사라지고, 이제 색 추출은 아래 스포이드 토글을 통해서만 명시적으로 일어난다.

## 4. 스포이드 — 명시적 토글 + 3중 페널티

**동작**: "스포이드" 토글 버튼을 누르면 스포이드 모드가 켜지고, 그 상태에서 배경 영역을 탭하면 그 지점의 배경 픽셀 색을 추출해 `state.color`에 반영한 뒤 자동으로 모드가 꺼진다. 인형 실루엣이 아닌 **배경**에서만 샘플링한다(붓칠은 인형 실루엣 클리핑 안에서만 일어나므로 영역이 자연히 구분된다).

**페널티 3종** (스테이지 저장 시 `eyedropperUses` 카운트를 문서에 함께 저장):
1. **횟수 제한**: 한 스테이지 제작당 최대 3회. 3회를 쓰면 토글 버튼이 비활성화된다.
2. **등급 상한**: `eyedropperUses > 0`이면 `computeGrade()`가 산정한 등급이 S라도 **A로 강제 하향**한다(그 외 등급은 그대로).
3. **정직 배지**: `eyedropperUses > 0`이면 "🧪 스포이드 사용됨" 배지를 다음 세 곳에 표시한다 — ① 찾는 사람의 결과 화면(`playend` 패널) ② 공유 카드(1080×1080 canvas, 등급 배지 옆) ③ 제작자 전용 결과 보기 화면(`result-panel`).

## 5. 설정값 변경

- `CONFIG.SCALE_MIN`: `0.04` → `0.10` (최소 인형 크기를 10%로 상향, 최대 `0.20`은 유지). 배치 화면 슬라이더의 `min` 속성도 함께 수정.
- 새 상수: `EYEDROPPER_MAX_USES: 3`, `EYEDROPPER_GRADE_CAP: 'A'`, 크롭 배경의 여백 배율(예: `PAINT_BG_CROP_RATIO: 1.6`).

## 6. 데이터 모델 / 보안 규칙

- `stages/{id}` 문서에 필드 추가: `eyedropperUses: number` (0~3).
- `firestore.rules`의 `allow create` 조건에 `request.resource.data.eyedropperUses is int && request.resource.data.eyedropperUses >= 0 && request.resource.data.eyedropperUses <= 3` 검증을 추가하고, `keys().hasAll([...])` 목록에도 포함시킨다.
- 기존 문서 크기 예산(photo ≤700KB + doll ≤150KB + taps ~20KB < 1MB)에는 영향 없음 — 정수 필드 하나 추가는 무시할 수준.

## 7. 에러 처리 / 엣지 케이스

- 사진이 너무 작아서(예: 인형 크기 대비) 크롭 영역이 사진 경계를 벗어나는 경우: 크롭 좌표를 사진 범위 안으로 clamp하고, 부족한 부분은 캔버스 배경색(도화지 색)으로 채운다.
- 배치를 안 바꾸고 칠하기 화면을 여러 번 드나들어도 배경 재캡처는 매번 최신 배치 기준으로 이뤄지므로 항상 정확하다.
- 스포이드 3회를 다 쓴 상태에서 "다시 배치"로 돌아갔다가 다시 칠하기로 와도 `eyedropperUses` 카운트는 유지된다(저장 전까지는 세션 로컬 상태).

## 8. 테스트 계획

- 확대 크롭 배경이 배치 각도/좌우반전과 무관하게 항상 고정(비회전)으로 보이고, 인형만 올바른 각도로 그려지는지 육안 확인.
- 네이티브 색상 피커로 고른 색이 브러시에 정확히 반영되는지 확인.
- 스포이드 토글 → 배경 탭 → 색 추출 → 자동 꺼짐 → 카운트 +1 흐름 확인, 4회째 시도 시 버튼 비활성화 확인.
- `eyedropperUses > 0`으로 저장한 스테이지가 실제로 S 등급을 못 받고 A로 캡되는지, 배지가 3곳 모두에 표시되는지 확인.
- `eyedropperUses = 0`인 기존 로직(스포이드 미사용) 스테이지는 기존과 동일하게 S까지 등급을 받을 수 있는지 회귀 확인.
- 최소 크기 10% 슬라이더 하한 확인.
