# 인형 최대 3개 배치 기능 설계

> 작성일: 2026-07-21
> 관련 백로그: 없음(신규). CLAUDE.md 백로그의 "미끼 인형 2개"(가짜 실루엣)와는 다른 기능 — 여기서는 실제로 찾아야 하는 진짜 인형이 여러 개다.

## 배경 / 목표

현재 스테이지에는 인형을 딱 하나만 숨길 수 있다. 제작자가 원하면 1~3개까지 자유롭게 배치할 수 있게 해, 도전자가 여러 개를 모두 찾아야 클리어되는 더 어려운 스테이지를 만들 수 있게 한다.

## 핵심 게임 디자인 결정

- **승리 조건**: 배치된 인형을 **전부** 찾아야 클리어
- **인형 개수**: 제작자가 1~3개 중 자유 선택 (기존 단일 인형 스테이지는 자연스럽게 "1개" 케이스)
- **제작 순서**: 인형별로 "배치 → 색칠"을 반복 (인형 1 배치→색칠, 필요하면 인형 2 배치→색칠, ...)
- **힌트 대상**: 사분면·온도계·반짝 3종 모두 **가장 가까운 미발견 인형**을 기준으로 동작
- **힌트 횟수**: 종류당 총 사용 횟수 = 인형 개수 (플레이 전체에서 소진되면 끝, 인형별로 재충전되지 않음)
- **스포이드 예산**: 인형별로 각각 최대 3회 (인형이 3개면 총 9회 가능하지만, 인형마다 독립적인 예산)
- **제한 시간**: 인형 개수와 무관하게 항상 60초 고정 (인형이 많을수록 자연히 더 어려워지는 것을 의도함)
- **발견 진행 표시**: 인형을 하나 찾을 때마다 타이머는 멈추지 않고 계속 진행, 기존 낙하 연출 + 진동을 그대로 재생하며 상태 문구를 "N/M 찾음"으로 갱신. 전부 찾으면 그 시점에 클리어
- **하위 호환**: 기존에 저장된 단일 인형 스테이지는 관련 코드를 전혀 건드리지 않고 기존 그대로 동작. 이번 기능은 `dolls` 배열이 있는 **신규 스테이지에서만** 적용되는 별도 코드 경로로 추가한다 (기존 `doll` 단일 필드 스테이지용 코드는 무변경)

## 데이터 모델

### 신규 스테이지 (다중 인형)
```
dolls: [
  { x, y, scale, rotation, flip, dollB64, eyedropperUses },  // 인형 1
  { x, y, scale, rotation, flip, dollB64, eyedropperUses },  // 인형 2 (있으면)
  { x, y, scale, rotation, flip, dollB64, eyedropperUses },  // 인형 3 (있으면)
]
```
- 배열 길이 1~3
- 각 원소의 필드 의미는 기존 `doll`/`dollB64`/`eyedropperUses`와 동일 (개별화만 됨)
- 문서에는 `dolls` 필드만 존재하고 `doll`/`dollB64`/`eyedropperUses` 단일 필드는 쓰지 않음

### 기존 스테이지 (단일 인형, 무변경)
```
doll: { x, y, scale, rotation, flip }
dollB64: "..."
eyedropperUses: N
```
- 그대로 유지. 마이그레이션하지 않는다.

### 로드 시 분기
`loadStage()`(플레이용)와 제작자 결과 보기 로드 로직에서, 문서에 `dolls` 배열이 있으면 신규 다중 인형 경로로, 없으면(=`doll` 단일 필드만 있으면) 기존 코드 경로로 그대로 처리한다. 두 경로는 이번 작업에서 통합하지 않는다.

### CONFIG 추가
```js
MAX_DOLLS: 3,
```

### Firestore 보안 규칙 추가
- `dolls`가 존재할 경우: `dolls.size() >= 1 && dolls.size() <= 3`
- `dolls`의 각 원소에 대해 기존 `dollB64` 크기 제한(150KB 초과 시 192px 축소 재시도 로직은 클라이언트 그대로, 규칙에서도 원소별 크기 검증)을 반복 적용
- 기존 `doll`/`dollB64` 검증 규칙은 무변경으로 유지 (기존 스테이지 대응)

## 제작 흐름 (배치 화면)

- 배치 패널 상단에 번호 탭(①②③) — 탭 개수 = 현재 배치된 인형 수
- 탭을 누르면 해당 인형이 "활성 인형"이 되어 하단 크기/회전/좌우반전 슬라이더가 그 인형을 조작
- 마지막 탭 옆에 "+ 인형 추가" 버튼 — 인형이 3개 미만일 때만 노출. 누르면 사진 중앙에 기본값(크기 10%, 회전 0°)으로 새 인형을 추가하고 그 탭이 자동 활성화됨
- 각 탭에 삭제(✕) 표시 — 인형이 2개 이상일 때만 노출 (최소 1개는 항상 유지, 마지막 하나는 삭제 불가)
- 비활성 인형들은 사진 위에 참고용으로 함께 렌더링됨(각자의 현재 합성 결과 그대로 — 아직 색칠 전이면 흰 실루엣, 색칠했으면 그 색). 드래그는 활성 인형만 가능(기존 근접 판정 로직 그대로, 비활성 인형에는 적용 안 함)
- "칠하러 가기" 버튼은 활성 인형 기준으로 색칠 화면 진입

## 제작 흐름 (색칠 화면)

- 확대 기능 포함 기존 색칠 로직 전부 활성 인형 기준으로 그대로 동작(캔버스·스포이드·붓 등 인형별로 독립된 상태를 가짐 — 내부적으로 인형마다 별도의 `paintCanvas`/`dollComposite`/`bgCropCanvas` 오프스크린 캔버스 세트를 유지)
- 색칠 완료 후 패널 액션에 "+ 인형 추가" 버튼 추가(3개 미만일 때만 노출) — 누르면 배치 화면으로 돌아가 새 인형을 배치
- 스포이드는 활성 인형의 `eyedropperUses` 필드에 개별 기록, 최대 3회

## 플레이 로직

- 탭할 때마다 아직 못 찾은 모든 인형에 대해 기존 원 판정 로직을 반복 검사, 하나라도 맞으면 그 인형만 "발견" 처리
- 발견 시: 기존 팔랑 낙하 연출 + 진동을 그 인형에 대해 재생, 상태 문구를 "N/M 찾음"으로 갱신, 해당 인형은 이후 판정/힌트 대상에서 제외
- 전부 찾으면 그 시점에 타이머 정지, 클리어 처리(기존 소요시간 기록 로직 그대로, 값은 "전부 찾을 때까지" 시간)
- 시간 초과 시: 그때까지 못 찾은 인형들만 위치 공개(기존 점선 링 펄스 연출을 인형마다 개별 재생)

## 힌트 3종

- 사분면(🧭)·온도계(🌡️)·반짝(✨) 모두 "현재 가장 가까운 미발견 인형"을 매 힌트 사용 시점에 다시 계산해 대상으로 삼음 (인형을 하나 찾으면 다음 힌트는 자동으로 다음으로 가까운 인형 기준이 됨)
- 온도계는 이후 매 탭마다 거리 피드백을 주는데, 이때도 "가장 가까운 미발견 인형까지의 거리"를 기준으로 계산
- 각 힌트 종류당 총 사용 가능 횟수 = 인형 개수 (플레이 시작 시 결정, 인형을 찾아도 재충전되지 않음 — 예: 인형 3개 스테이지는 사분면 힌트를 플레이 전체에서 최대 3번 쓸 수 있음)

## 결과 화면 · 등급 · 히트맵

- 소요시간은 "전부 찾을 때까지 걸린 시간"으로 기존과 동일한 필드(`totalClearMs` 등)에 기록
- 위장 등급 계산(평균 발견시간/제한시간 비율, `computeGrade()`)은 로직 변경 없이 그대로 재사용 — 인형 수와 무관하게 스테이지별 통계로 자연히 반영됨
- 정직 배지(스포이드 사용 표시 + 최고등급 A 제한)는 인형 중 하나라도 스포이드를 썼으면(=배열 원소 중 `eyedropperUses > 0`이 하나라도 있으면) 스테이지 전체에 적용 — 판단 기준만 배열 순회로 바뀌고 로직은 동일
- 탭 히트맵은 기존처럼 오답/정답 구분 없이 전체 탭을 하나의 히트맵에 표시. 제작자 결과 보기에서 정답 위치는 배치된 인형 전부를 각자의 위치에 표시

## 공유 카드 · 갤러리

- 공유 카드 문구: 인형이 2개 이상이면 "숨은 인형 N마리를 찾아라!" 형태로 개수 표시, 1개면 기존 문구 그대로 유지
- 홈 예제 갤러리 카드: 인형 개수가 2개 이상인 스테이지에만 작은 개수 뱃지 추가, 1개면 기존과 동일하게 뱃지 없음

## 영향 범위 요약

- `index.html` 단일 파일 내 변경만 발생
- 기존 단일 인형 스테이지 관련 함수/로직은 무변경 (읽기 전용 하위호환)
- 신규 `dolls` 배열 경로를 위한 새 함수/분기 추가: 제작(배치 탭·추가·삭제), 색칠(인형별 오프스크린 캔버스 세트), 플레이(다중 판정·다중 힌트 타겟팅·다중 발견 연출), 결과(다중 정답 표시), 공유 카드 문구, 갤러리 뱃지
- `CONFIG.MAX_DOLLS` 추가, 그 외 `CONFIG` 상수는 무변경(등급 구간, 힌트 로직 등 기존 값 재사용)
- `firestore.rules`에 `dolls` 배열 검증 규칙 추가 (기존 `doll` 검증 규칙과 병행 유지)
