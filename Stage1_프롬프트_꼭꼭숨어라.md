# 「꼭꼭 숨어라」 Stage 1 — Claude Code 프롬프트

> 3단계 구성 기준: **Stage 1** 에디터+배치+로컬 플레이 / Stage 2 Firebase 연동+플레이 화면 / Stage 3 결과·등급·히트맵·마감
> 아래 구분선 이하 전체를 Claude Code에 붙여넣으세요.

---

너는 시니어 프론트엔드 개발자야. 「꼭꼭 숨어라」라는 웹 게임의 Stage 1을 구현한다.

## 0. 시작 전 필수 작업

프로젝트 루트에 `CLAUDE.md`를 먼저 생성하고, 이후 모든 작업 전에 이 파일을 읽는 것을 원칙으로 한다. `CLAUDE.md`에는 다음을 기록한다:

- 프로젝트 한 줄 정의: "내 사진에서 색을 훔쳐 종이인형을 위장시키고, 친구가 그걸 찾아내는 비동기 숨바꼭질 웹앱"
- 개발 규약 (아래 1번 전체)
- 3단계 로드맵: Stage 1 에디터+배치+로컬 플레이 → Stage 2 Firebase 연동(익명 인증, 리사이즈 업로드, `?s=` URL 로드, 삭제키)+정식 플레이 화면(카운트다운·타이머·발견 연출) → Stage 3 결과 화면·위장 등급·탭 히트맵·예제 스테이지 5개·마감
- 현재 진행 상태 (Stage 1 완료 시 완료 항목 체크 후 갱신)

## 1. 개발 규약 (절대 준수)

- **단일 파일 `index.html`** — 빌드 도구 없음, 프레임워크 없음, 순수 HTML/CSS/JS
- 외부 의존성은 CDN만 허용 (Stage 1에서는 Google Fonts의 Gaegu 폰트만 사용)
- 모든 상수는 파일 최상단 `CONFIG` 객체에 모은다
- 주석은 한국어로 작성한다
- 배포 대상은 GitHub Pages (Stage 2에서 `?s=` 쿼리 파라미터 라우팅 예정이므로 화면 전환은 JS로 섹션 show/hide 방식)
- 디자인 토큰: 도화지 화이트 `#FAF7F0`, 연필 그레이 `#4A4A4A`, 포인트 레드 `#E45B5B`, 폰트 Gaegu — 손으로 만든 종이 공작 감성 (부드러운 그림자, 살짝 기울어진 카드, 손그림 느낌 테두리)
- **모든 화면 최하단에 공통 푸터 `made by JSTD`를 표기한다** (작은 글씨, 링크 없음)
- 모바일 퍼스트: 터치 이벤트 우선, 최소 탭 영역 44px, 세로 화면 기준 레이아웃

## 2. Stage 1 구현 범위

### A. 화면 구조 (섹션 3개 + 공통 푸터)
1. **홈 화면**: 타이틀, "숨기러 가기" 버튼 (예제 갤러리 자리는 placeholder로 비워둠 — Stage 3에서 채움)
2. **제작 화면**: 사진 업로드 → 인형 색칠 → 배치 → 테스트 플레이
3. **테스트 플레이 화면**: 자신이 숨긴 인형을 직접 찾아보는 로컬 검증용

### B. 사진 업로드 처리
- `<input type="file" accept="image/*">` + 카메라 촬영 허용
- 업로드 즉시 클라이언트에서 처리:
  1. EXIF orientation 값을 읽어 회전 보정 (아이폰 세로 사진이 눕는 버그 방지 — ArrayBuffer에서 orientation 태그 직접 파싱, 라이브러리 없이 구현)
  2. canvas로 긴 변 1280px 리사이즈, JPEG 품질 0.8 재인코딩 (EXIF는 이 과정에서 자동 제거됨)
  3. 결과는 메모리(dataURL/Blob)에만 보관 — Stage 1은 서버 저장 없음
- 처리 완료 후 사진을 제작 화면 배경 캔버스에 표시

### C. 인형 색칠 에디터 (핵심 기능 — 가장 공들일 것)
- 흰 종이인형 실루엣이 그려진 256×256 편집 캔버스 (인형 모양: 심플한 사람형 실루엣을 SVG path로 정의 후 캔버스에 렌더, 인형 외곽선은 연필 느낌의 얇은 회색 선)
- 색칠은 인형 실루엣 내부에만 적용 (클리핑 패스 사용, 바깥으로 번지지 않게)
- 도구:
  - **스포이드**: 배경 사진의 아무 지점을 탭하면 그 픽셀 색을 현재 브러시 색으로 추출 (`getImageData`). 현재 색은 팔레트 원으로 항상 표시. **이 앱의 핵심 메커니즘이므로 사진 탭 → 색 추출 → 즉시 칠하기 흐름이 끊기지 않게 UX 설계**
  - 브러시 3종 (대 24px / 중 12px / 소 5px), 부드러운 원형 브러시
  - 지우개 (칠한 부분만 지움, 실루엣과 외곽선은 유지)
  - 실행취소 10단계 (캔버스 스냅샷 스택)
  - 전체 지우기 (확인 후)
- 편집 캔버스는 배경 사진 위에 반투명 오버레이 패널로 띄워, 칠하면서 사진 색과 비교 가능하게 배치

### D. 인형 배치
- 색칠 완료 후 "숨기기" 버튼 → 인형이 사진 위 드래그 가능한 오브젝트로 전환
- 한 손가락 드래그 이동 / 크기 슬라이더 (사진 짧은 변의 4%~20%) / 회전 슬라이더 (0~360°) / 좌우반전 토글
- 인형의 위치·크기·회전·반전 상태는 **사진 기준 정규화 좌표(0~1)**로 상태 객체에 저장 (Stage 2에서 Firestore 스키마와 직결되므로 구조 준수):
  ```js
  doll: { x, y, scale, rotation, flip }
  ```
- 사진과 인형은 절대 하나의 이미지로 합성(굽기)하지 않는다 — 항상 별도 레이어로 렌더

### E. 테스트 플레이 (로컬 판정 검증)
- "테스트 플레이" 버튼 → 편집 UI 숨기고 사진+인형만 표시
- 탭 판정: 탭 좌표(정규화) vs 인형 바운딩 박스 + 여유 반경(박스 크기의 15%). 인형이 작을 경우 최소 판정 반경(사진 짧은 변의 3%) 보장
- 오답: 탭 지점에 잔물결 애니메이션 / 정답: 인형이 살짝 커졌다가 팔랑 떨어지는 발견 연출(CSS 애니메이션) + "찾았다!" 표시
- "다시 편집" 버튼으로 제작 화면 복귀 (상태 유지)

## 3. 완료 기준 (모두 충족 시 Stage 1 종료)

1. 아이폰 세로 사진 업로드 시 방향이 올바르게 표시된다
2. 스포이드로 사진 색을 추출해 인형을 칠하고, 실행취소가 10단계 동작한다
3. 인형을 드래그·크기·회전·반전 배치하고 테스트 플레이에서 정답/오답 판정이 정확하다
4. 브라우저 창 크기를 바꿔도 정규화 좌표 덕분에 인형 위치가 사진 기준으로 유지된다
5. 모든 화면 하단에 `made by JSTD` 푸터가 보인다
6. `CLAUDE.md`가 생성되어 있고 Stage 1 완료 상태가 기록되어 있다
