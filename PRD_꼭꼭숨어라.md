# PRD — 「꼭꼭 숨어라」 (가칭)
### 사진 속에 종이인형을 숨기는 비동기 숨바꼭질 웹앱

- 버전: v1.0 (초안)
- 작성일: 2026-07-18
- 목표: **작지만 완성도 높은 단일 파일 웹앱** (혼자 개발, AI API 미사용, Firebase 무료 플랜)
- 영감: Steam 'MECCHA CHAMELEON' — 단, 2D 흉내가 아닌 독자적 재미 구축

---

## 1. 한 줄 정의

> **"내 사진에서 색을 훔쳐 종이인형을 위장시키고, 친구가 그걸 찾아내는 비동기 숨바꼭질."**

- 제작자(숨기는 사람)가 사진을 올리고, 흰 종이인형을 **사진에서 직접 추출한 색**으로 칠해 숨긴다.
- 저장하면 고유 URL이 생성되고, 링크를 받은 사람은 **제한 시간 안에** 인형을 찾는다.
- 게임이 끝나면 제작자는 "사람들이 어디를 눌렀는지" 히트맵과 위장 등급을 피드백으로 받는다.

## 2. 차별화 전략 — "메차 카멜레온 2D 흉내"가 되지 않기 위한 3가지 축

원작은 **찾는 사람**의 게임이다. 이 앱은 축을 뒤집어 **숨기는 사람을 아티스트로** 만든다.

| 축 | 내용 | 원작과의 차이 |
|---|---|---|
| ① 위장이 곧 스킬 | **스포이드 도구**로 사진의 실제 픽셀 색을 추출해 칠한다. 잘 칠할수록 정말 안 보인다. 색칠 실력 = 게임 난이도 | 원작은 미리 만들어진 3D 씬. 여기선 위장 제작 자체가 창작 플레이 |
| ② 제작자 보상 루프 | 플레이 결과가 제작자에게 돌아온다: 탭 히트맵, 평균 발견 시간, 위장 등급(S~D) | 원작에는 없는 UGC 피드백 루프. "다음엔 더 잘 숨겨야지"라는 반복 동기 |
| ③ 종이인형 감성 | 흰 도화지 인형 + 손그림 아웃라인 + 발견 시 팔랑~ 떨어지는 애니메이션. 디지털 위장이 아닌 **아날로그 공작 감성** | 원작의 3D 카멜레온과 완전히 다른 미학. 앱의 시각 아이덴티티 |

## 3. 기획 분석

### 3-1. 장점
- **비동기 구조**: 실시간 서버·매칭 불필요. URL 공유만으로 성립 → 무료 플랜에 최적.
- **UGC 이중 재미**: 만드는 재미(색칠·숨기기)와 찾는 재미가 분리되어 둘 다 콘텐츠가 된다.
- **개인적인 사진**: "우리 교실", "우리 집 책상" 등 아는 장소일수록 재미가 커진다 → 친구·가족·학급 단위 바이럴에 유리.
- **낮은 진입 장벽**: 설치 없음, 로그인 없음(익명 인증), 링크 하나로 플레이.

### 3-2. 약점과 대응
| 약점 | 대응 |
|---|---|
| 1회성 콘텐츠 (한 번 찾으면 끝) | 히트맵·등급 피드백으로 **제작 반복** 유도, 결과 공유 카드로 재확산, 내장 예제 스테이지 |
| 부정행위 (스크린샷 비교, 확대) | 인형을 사진에 굽지 않고 **캔버스 오버레이로 실시간 합성** → 원본 이미지 저장해도 인형 없음. 확대는 "돋보기 2회 제한" 기능으로 게임 규칙화 |
| 콜드 스타트 (초기 콘텐츠 부재) | 공식 예제 스테이지 5개 내장(첫 방문자가 바로 플레이 가능) |
| 사진 프라이버시 | EXIF(GPS 포함) 완전 제거, 기본 비공개(URL 아는 사람만), 신고 버튼, 삭제 키 제공 |
| 색칠이 어렵게 느껴질 수 있음 | 스포이드 + "주변색 자동 채우기" 버튼 1개 제공(반자동), 브러시 3종으로 단순화 |

### 3-3. 추가 재미 요소 (우선순위 포함)
- **[P0] 탭 히트맵**: 탐색자가 누른 지점을 모두 기록, 제작자 결과 화면에 반투명 점으로 표시.
- **[P0] 위장 등급**: 평균 발견 시간 기반 S/A/B/C/D 등급 + 한 줄 평 ("카멜레온도 울고 갈 위장!").
- **[P1] 미끼 인형**: 제작자가 가짜 실루엣(누르면 "속았지?" 연출)을 최대 2개 배치.
- **[P1] 힌트 3종**: ① 사분면 표시 ② 뜨겁다/차갑다 ③ 0.5초 아웃라인 반짝 — 사용 시 기록에 별표(패널티).
- **[P1] 결과 공유 카드**: "23초 만에 발견! (힌트 0회)" 이미지 카드 생성 → 인스타/카톡 공유.
- **[P2] 스테이지 이어달리기**: 클리어 후 "나도 숨기기" 버튼으로 제작 화면 직행 → 체인 형성.
- **[P2] 인형 포즈 3종**: 서있기/앉기/매달리기 — 배치 자유도 상승.

### 3-4. 기술적 주의점 (Firebase 무료 플랜 기준)
1. **업로드 전 클라이언트 리사이즈 필수**: canvas로 긴 변 1280px, JPEG 품질 0.8 → 장당 약 150~300KB. Storage 5GB로 약 2만 장 수용.
2. **인형은 사진과 분리 저장**: 인형은 256×256 PNG(투명 배경)로 별도 저장, 위치·크기·회전은 Firestore에 **정규화 좌표(0~1)**로 저장. → 기기 해상도 무관 판정 + 발견 애니메이션 가능 + 굽기 방지.
3. **EXIF 제거**: canvas 재인코딩 시 자동 제거되지만, **orientation 값을 먼저 읽어 회전 보정** 후 그릴 것(아이폰 세로 사진 90도 눕는 버그 방지).
4. **판정 로직**: 탭 좌표(정규화) vs 인형 바운딩 박스 + 여유 반경(박스 크기의 15%, 모바일 터치 오차 보정). 인형이 매우 작을 경우 최소 판정 반경 보장(짧은 변의 3%).
5. **라우팅**: GitHub Pages는 SPA 라우팅 불가 → `index.html?s={stageId}` 쿼리 파라미터 방식.
6. **인증·보안 규칙**: 익명 인증(signInAnonymously). Firestore 규칙 — 스테이지 read는 공개, write는 생성자 uid만. 플레이 기록은 append-only. 삭제는 생성 시 발급되는 `deleteKey` 일치 시에만.
7. **동시성**: 플레이 통계 갱신은 last-write-wins 금지 — `increment()` 연산자 사용 (mindpullness 데이터 손실 교훈 반영).
8. **히트맵 데이터 상한**: 스테이지당 탭 기록 최근 500개까지만 저장(배열 슬라이스), 문서 1MB 제한 대비.

## 4. 핵심 게임 루프

### 제작자 플로우
```
사진 선택 → 자동 리사이즈·EXIF 제거 → 인형 캔버스 색칠
(스포이드/브러시/자동채움/실행취소) → 사진 위 드래그 배치
(크기·회전·좌우반전) → 미리보기(찾기 테스트) → 저장
→ URL + 삭제키 발급 → 공유
```

### 탐색자 플로우
```
URL 접속 → 카운트다운 3초 → 제한 시간(기본 60초) 내 탭으로 탐색
→ 오답: 잔물결 + 탭 기록 / 힌트 사용 가능(3종)
→ 정답: 인형이 팔랑 떨어지는 발견 연출 + 기록 표시
→ 결과 화면(시간·힌트·등급) → 공유 카드 / "나도 숨기기"
```

## 5. 기능 명세

### P0 (MVP — 이것만으로 완결)
| 기능 | 상세 |
|---|---|
| 사진 업로드 | 파일 선택/카메라, 리사이즈 1280px·JPEG 0.8, orientation 보정, EXIF 제거 |
| 인형 에디터 | 흰 종이인형 캔버스(256×256), 브러시(대/중/소), **스포이드(사진 색 추출)**, 지우개, 실행취소 10단계, 전체 지우기 |
| 배치 | 드래그 이동, 핀치/슬라이더 크기(사진 짧은 변의 4%~20%), 회전, 좌우반전 |
| 저장·공유 | Storage(photo.jpg, doll.png) + Firestore(메타), URL·삭제키 발급, 클립보드 복사 |
| 플레이 | 타이머 60초, 탭 판정, 오답 잔물결, 정답 발견 연출, 실패 시 위치 공개 |
| 결과 | 소요 시간, 히트맵 기록 저장(`increment` 사용), 등급 산정 |
| 예제 스테이지 | 내장 5개(첫 화면 갤러리) |
| **푸터** | **앱 최하단에 `made by JSTD` 푸터를 반드시 표기한다 (모든 화면 공통, 작은 글씨, 링크 없음)** |

### P1 (완성도)
힌트 3종 / 미끼 인형 2개 / 제작자용 히트맵 결과 화면 / 공유 카드 이미지 생성(canvas) / 타이머 옵션(30·60·120초)

### P2 (확장)
인형 포즈 3종 / 이어달리기 체인 / 스테이지 공개 갤러리(신고 기능 포함)

## 6. 데이터 모델

```
Firestore
└─ stages/{stageId}
   ├─ createdAt, creatorUid, deleteKeyHash
   ├─ photoPath, dollPath          // Storage 경로
   ├─ doll: { x, y, scale, rotation, flip }   // 모두 0~1 정규화
   ├─ decoys: [{ x, y, scale }]    // P1
   ├─ timeLimit: 60
   ├─ stats: { plays, clears, totalClearMs }  // increment()로만 갱신
   └─ taps: [ { x, y } ]           // 최근 500개 슬라이스

Storage
└─ stages/{stageId}/photo.jpg (≤300KB), doll.png (≤50KB)
```

**등급 산정**: 평균 발견 시간 비율 = totalClearMs / clears / timeLimit
→ 80%↑ S / 60%↑ A / 40%↑ B / 20%↑ C / 미만 D (클리어 0회면 "미개봉 위장").

## 7. 기술 스택·규약 (정우 표준 워크플로우)

- **단일 파일 `index.html`**, 빌드 도구 없음, CDN만 사용(Firebase compat SDK)
- 상수는 최상단 `CONFIG` 객체, 주석은 한국어
- 배포: GitHub Pages, 라우팅은 `?s=` 쿼리 파라미터
- 폰트: Gaegu(손글씨 감성, 포트폴리오 사이트와 통일)
- 색: 도화지 화이트 `#FAF7F0`, 연필 그레이 `#4A4A4A`, 포인트 레드 `#E45B5B`

## 8. Claude Code 단계별 구현 계획

각 단계 시작 시 **`CLAUDE.md`를 먼저 읽는다**. CLAUDE.md에는 본 PRD 요약, 규약(단일 파일·CONFIG·한국어 주석), 현재 단계 상태를 기록한다.

| 단계 | 산출물 | 완료 기준 |
|---|---|---|
| 1 | 정적 셸 + 인형 색칠 에디터 (로컬) | 사진 미리보기 위에서 스포이드로 색 추출해 인형 색칠, 실행취소 동작. **푸터 `made by JSTD` 포함** |
| 2 | 배치 + 판정 (로컬 플레이) | 드래그·크기·회전 배치 후 "테스트 플레이"로 자기 인형 찾기 성공/실패 판정 |
| 3 | Firebase 연동 | 익명 인증, 리사이즈 업로드, 스테이지 저장, `?s=` URL로 로드, 삭제키 동작, 보안 규칙 배포 |
| 4 | 플레이 화면 완성 | 카운트다운, 타이머, 오답 잔물결, 발견 애니메이션, 실패 공개, stats `increment` |
| 5 | 결과·마감 | 등급·히트맵 기록, 결과 화면, 예제 스테이지 5개, 모바일 반응형 점검, 푸터 전 화면 확인 |

## 9. 비기능 요구사항·리스크

- **성능**: 첫 로드 3초 이내(사진 lazy load), 캔버스 조작 60fps 목표(터치 이벤트 passive 처리)
- **접근성**: 최소 탭 영역 44px, 색약 대비 — 힌트 ② "뜨겁다/차갑다"는 색이 아닌 텍스트+진동으로 표시
- **콘텐츠 안전**: 사진은 기본 비공개(URL 소지자만), 공개 갤러리(P2)는 신고 누적 3회 시 자동 숨김
- **최대 리스크**: 색칠 UX가 모바일에서 답답하면 핵심 재미가 붕괴 → **1단계에서 실기기 테스트 최우선**, 브러시는 3종으로 제한해 단순함 유지
