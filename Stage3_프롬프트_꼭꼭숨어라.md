너는 시니어 프론트엔드 개발자야. 「꼭꼭 숨어라」의 마지막 단계인 Stage 3을 구현하고 프로젝트를 마감한다.

## 0. 시작 전 필수 작업

1. **`CLAUDE.md`를 먼저 읽고** 규약과 Stage 1~2 완료 상태를 파악한다.
2. `index.html`의 플레이 종료 흐름, stats/taps 기록 로직, 임시 결과 표시 부분을 확인한 뒤 작업한다.
3. 완료 시 `CLAUDE.md`를 최종 상태로 갱신하고, 미구현 확장 아이디어를 "백로그" 섹션에 기록한다.

## 1. 규약 재확인 (절대 준수)

- 단일 파일 `index.html`, 순수 HTML/CSS/JS, CDN만 허용, `CONFIG` 객체, 한국어 주석
- 모든 화면 최하단 공통 푸터 `made by JSTD` 유지
- stats 갱신은 `increment()` 전용, 사진·인형 합성 금지, 외부 그림 라이브러리 금지

## 2. Stage 3 구현 범위

### A. 힌트 시스템 3종 (플레이 화면)
- 플레이 화면 하단에 힌트 버튼 3개, 각 1회씩만 사용 가능:
  1. **사분면**: 사진을 4등분하는 점선이 1.5초 표시되고 인형이 있는 사분면이 살짝 밝아진다
  2. **뜨겁다/차갑다**: 사용 후부터 매 탭마다 인형과의 거리(정규화 거리 기준)를 "앗 뜨거!(0.1 미만) / 따뜻해(0.25 미만) / 미지근(0.45 미만) / 차가워(그 외)" **텍스트로** 표시 + `navigator.vibrate` 지원 시 거리별 진동 패턴 (색으로만 구분하지 않는다 — 색약 접근성)
  3. **반짝**: 인형 아웃라인이 0.5초 동안 한 번 반짝인다
- 사용한 힌트 수는 플레이 기록에 포함하고 결과 화면에 별표(★)로 표시
- 힌트 사용 여부는 스테이지 stats에 저장하지 않는다 (개인 기록 전용)

### B. 정식 결과 화면 (Stage 2의 임시 결과 표시를 대체)
- 성공 시: 소요 시간(초 단위 소수 1자리), 사용 힌트 수, 남은 시간 비율 게이지
- 실패 시: "다음엔 찾을 수 있어!" + 인형 위치 공개(기존 연출 유지)
- **스테이지 위장 등급** 표시 (해당 스테이지의 누적 stats 기반):
  ```
  비율 = stats.totalClearMs / stats.clears / (timeLimit × 1000)
  0.8 이상 S / 0.6 이상 A / 0.4 이상 B / 0.2 이상 C / 미만 D
  clears가 0이면 "미개봉 위장" 표시
  ```
  등급별 한 줄 평 예시 (CONFIG에 문구 배열로): S "카멜레온도 울고 갈 위장!" / A "매의 눈도 헤맸다" / B "제법인데?" / C "살짝 보였어" / D "숨은 그림 아니고 그냥 그림"
- 버튼: "결과 카드 공유" / "나도 숨기기"(제작 화면 직행) / "홈으로"

### C. 공유 카드 생성 (canvas)
- "결과 카드 공유" 클릭 시 1080×1080 canvas에 카드 이미지 생성:
  - 도화지 배경 + 앱 타이틀 + "OO초 만에 발견! (힌트 ★N)" 또는 "찾지 못했다..." + 위장 등급 뱃지 + 하단에 작게 `made by JSTD`
  - **사진과 인형은 카드에 넣지 않는다** (스포일러 + 프라이버시 방지) — 종이인형 실루엣 일러스트만 장식으로 사용
- Web Share API(`navigator.share`, files 지원 확인) 우선, 미지원 시 PNG 다운로드 폴백

### D. 제작자 결과 보기 — 탭 히트맵
- 저장 완료 화면에 플레이 URL과 별도로 **결과 보기 URL** 추가: `?s={stageId}&view=result`
  - "이 링크는 정답 위치가 보이니 제작자만 간직하세요" 안내 문구
- 결과 보기 화면 구성:
  - 사진 위에 인형을 정답 위치에 표시
  - `taps` 배열의 모든 탭을 반투명 포인트 레드 원(겹칠수록 진해짐)으로 오버레이 → 히트맵
  - stats 요약: 플레이 수 / 클리어 수 / 평균 발견 시간 / 위장 등급
  - "스테이지 삭제" 링크(Stage 2 삭제 플로우 재사용)

### E. 홈 화면 예제 갤러리
- `CONFIG.EXAMPLE_STAGES = []` 배열에 스테이지 ID를 넣으면 홈 화면에 갤러리로 표시되는 구조 구현:
  - 각 ID의 Firestore 문서에서 photo.jpg 썸네일(블러 강하게 처리 — 정답 노출 방지)과 위장 등급을 로드
  - 카드 탭 시 해당 스테이지 플레이 시작
  - 배열이 비어 있으면 갤러리 섹션 자체를 숨긴다
- 예제 5개는 배포 후 제작자(정우)가 직접 만들어 ID를 채운다 — 주석으로 채우는 위치 안내

### F. 마감 점검 (전체 품질 패스)
1. **반응형**: 320px(구형 폰) ~ 1440px(데스크톱)에서 레이아웃 깨짐 없음, 데스크톱에서는 최대 폭 520px 중앙 정렬
2. **접근성**: 모든 버튼 탭 영역 44px 이상, 힌트②는 텍스트+진동(색 비의존), 주요 버튼에 `aria-label`
3. **로딩·에러**: 모든 비동기 구간에 스피너와 실패 시 재시도 UI가 있는지 전수 점검
4. **성능**: 첫 화면 로드 시 Firebase SDK를 제외한 이미지·데이터는 lazy load, 캔버스 터치 이벤트 passive 처리 확인
5. **푸터**: 모든 화면(홈·제작·플레이·결과·결과 보기·삭제·에러)에 `made by JSTD` 표시 전수 확인
6. `CLAUDE.md` 최종 갱신: Stage 1~3 완료 체크, 배포 절차(GitHub Pages) 메모, 백로그 기록(미끼 인형 2개 / 인형 포즈 3종 / 타이머 옵션 30·60·120초 / 이어달리기 체인 / 공개 갤러리+신고)

## 3. 완료 기준 (모두 충족 시 프로젝트 완성)

1. 힌트 3종이 각 1회 동작하고 결과 화면에 사용 수가 ★로 표시된다
2. 여러 번 플레이한 스테이지의 위장 등급이 공식대로 산정·표시된다 (clears 0이면 "미개봉 위장")
3. 결과 카드가 생성되어 모바일에서 공유 시트가 뜨거나 PNG로 저장된다 (사진·인형 미포함 확인)
4. 결과 보기 URL에서 히트맵과 stats 요약이 정확히 표시된다
5. `CONFIG.EXAMPLE_STAGES`에 ID를 넣으면 홈 갤러리가 나타나고, 비우면 사라진다
6. 마감 점검 6개 항목이 모두 통과되고 `CLAUDE.md`에 백로그까지 기록되어 있다

## 4. 하지 말 것

- 백로그 항목(미끼 인형, 포즈, 타이머 옵션, 체인, 공개 갤러리) 구현 금지 — 기록만 한다
- 히트맵을 위해 새 라이브러리 추가 금지 — 반투명 원 오버레이로 충분
- 공유 카드에 스테이지 사진이나 인형 위치가 드러나는 요소 포함 금지
- 기존 Stage 1~2 규약 위반 금지 (단일 파일, `increment()` 전용, 합성 금지)
